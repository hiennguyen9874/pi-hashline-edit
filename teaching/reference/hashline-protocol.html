Hashline Protocol Reference

Cheat sheet for the LINE#HASH│content protocol

Display format

LINE#HASH│content
LINE Display-only line number. Used for offset/limit navigation. Never included in edit/insert anchors.
# Anchor separator (ANCHOR_SEP).
HASH 3 characters from A-Z a-z 0-9 - _. Content-addressed, context-aware. Copy this into edit/insert.
│ Content separator (CONTENT_SEP, U+2502).

Hash alphabet

A-Z a-z 0-9 - _  (64 chars = 6 bits/char × 3 chars = 18 bits = 262,144 values)

Hash computation

hash = xxHash32(prev + "\0" + curr + "\0" + next  [+ ":R" + retry], seed=0)
       → extract top 18 bits → encode in base64url

Context: prev and next lines are included so identical lines in different positions get different hashes (within a 262K value space, collisions are resolved via retry).

Collision resolution: if a hash collides with another in the same file, retry with :R1, :R2, … until unique.

Anchor resolution (3 tiers)

Tier 1: Exact Hash matches uniquely in current file, position unchanged. partitionExact()
Tier 2: Fuzzy Hash matches uniquely but at different line. Relocate with [RELOCATED] warning. fuzzyMatch()
Tier 3: Merge Hash doesn't match current file. Try snapshot: apply edits to snapshot, diff, apply patch to current. threeWayMerge()

Error codes

[E_BAD_REF] Anchor is not a valid 3-character hash (includes line numbers, missing chars, etc.)
[E_STALE_ANCHOR] Hash doesn't exist in the current file (line deleted or heavily modified). Re-read.
[E_AMBIGUOUS_ANCHOR] Hash matches multiple lines. Re-read for fresh anchors.
[E_BARE_HASH_PREFIX] Edit payload contains hashline prefixes (HASH│content). Send literal file content.
[E_CURRENT_MISMATCH] Optional current field doesn't match the live line text.
[E_EDIT_CONFLICT] Two edits in the same request overlap on the same original line range.
[E_WOULD_EMPTY] Edit would delete entire file (>50 lines). Use write tool instead.

Key invariants

  • One perfect hash array per file. buildHashlineFile() is the single canonical computation.
  • Hash-only anchors. Mutating tools reject line-qualified anchors. Only the 3-char hash is accepted.
  • No silent fixup. The runtime rejects malformed input; it never autocorrects model errors.
  • Atomic writes. All file mutations go through writeFileAtomically() (temp file + rename).
